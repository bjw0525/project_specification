인증
Authentication

인증

Form Login 기반 인증 플로우와 리디렉션 동작을 정의해 사용자 경험을 단순화했습니다.

  • 로그인 페이지: /member/signin (GET/POST)
  • 로그아웃: /member/signout
  • 성공 시 /, 실패 시 /member/signin?error
Authorization

인가

URL 패턴 기반으로 공개/인증/관리자 전용 영역을 분리했습니다.

  • 공개: 홈, 정적 리소스, GET 공지/게시글/스토리
  • 인증 필요: 댓글 작성, 좋아요, 회원 정보 수정/삭제
  • ADMIN 전용: 게시글/스토리/공지 작성·수정·삭제, 관리자 페이지
Password Policy

비밀번호 인코딩

개발/운영 환경별 인코더 전략과 샘플 Bean 설정을 안내합니다.

  • 개발용: NoOpPasswordEncoder (평문) — 운영 금지
  • 운영 권장: BCryptPasswordEncoder
// 운영 전환 시 예시
@Bean
PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();
}

  • 유지 기간: 14일
  • Key: blog-remember-me-key
  • 쿠키 이름: remember-me (Spring 기본)
Hardening

추가 권장 사항

운영 안정성을 높이기 위한 보안 체크리스트입니다.

  • CSRF 토큰 활성화 및 SameSite 쿠키 적용
  • XSS 방어: 입력 검증 + 출력 인코딩 + CSP
  • SQL Injection 방지: 파라미터 바인딩, Mapper 검토
  • 파일 업로드 검증: 확장자·MIME·크기·경로 화이트리스트
  • API Rate Limiting, 감사 로깅, 보안 이벤트 모니터링
