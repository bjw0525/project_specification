보안 구성

Spring Security 기반으로 Form Login과 Remember-Me를 사용하며, URL 패턴별 접근 권한을 세분화합니다.

인증

  • 로그인 페이지: /member/signin (POST 처리 포함)
  • 로그아웃: /member/signout
  • 성공 시 /, 실패 시 /member/signin?error

인가

  • 공개: 홈, 정적 리소스, GET 공지/게시글/스토리 목록·상세
  • 인증 필요: 댓글 작성, 좋아요, 회원 정보 수정/삭제
  • ADMIN 전용: 게시글/스토리/공지 작성·수정·삭제, 관리자 페이지

비밀번호 인코딩

  • 개발용: NoOpPasswordEncoder (평문 저장) — 운영 금지
  • 운영 권장: BCryptPasswordEncoder
// 운영 전환 시 예시 (Java)
@Bean
PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();
}

Remember-Me

  • 유지 기간: 14일
  • Key: blog-remember-me-key

추가 권장 사항

  • CSRF 토큰 활성화
  • XSS 방어(입력 검증, 출력 인코딩), 컨텐츠 보안 정책(CSP) 고려
  • SQL Injection 방어(파라미터 바인딩, Mapper 검토)
  • 파일 업로드 검증(확장자, MIME, 크기, 저장 경로)
  • API Rate Limiting, 로깅/감사 전략 수립
